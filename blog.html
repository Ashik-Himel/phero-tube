Discuss the scope of var, let, and const

The variable that declared with var can have two scopes. They are, Function scope and Global scope. That means, if I declare a variable with var inside of function, it will be function scope and the variable cannot be accessible outside of that function. On the other hand, if I declare a variable with var outside of any function, it will be global scope and can be accessible from anywhere in the code.

The variable that declared with let and const can have two scopes also. They are, Global scope and Block scope. That means, if I declare a variable with let and const inside a block (inside curly brackets), it will be block scope and the variable cannot be accessible outside of that block. On the other hand, if I declare a variable with let and const outside of any block, it will be global scope and can be accessible anywhere from the code.

Tell us the use cases of null and undefined

We use null when we need to indicate the absence of value in our code. For example, if we need to initialize a variable that will be use to store data later, we can simply set the null value in that variable.

On the other hand, when we declare a variable without value, the value automatically set to undefined by default.

What do you mean by REST API?

REST API / RESTful API is an web API that conforms to the constraint of REST architectural style and allows for interaction with RESTful web services. REST stands for representational state transfer.
